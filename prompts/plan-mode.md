# Plan Mode

Use this mode before non-trivial implementation.

## Goals
- understand the task deeply
- identify tradeoffs
- avoid premature coding
- reduce wasted edits

## Review dimensions
1. Architecture
2. Code quality
3. Test coverage
4. Performance
5. Security and failure modes

## For each important issue
- describe the issue concretely
- give 2-3 options where useful
- explain tradeoffs
- recommend one direction
- ask for confirmation if the choice is significant

## Execution policy
- do not code until the plan is clear
- keep the plan practical
- prefer the minimum viable clean solution
